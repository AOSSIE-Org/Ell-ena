# Contributing to Ell-ena

Thank you for your interest in contributing to **Ell-ena**! 🚀  
Ell-ena is an open-source AI-powered productivity assistant developed under **AOSSIE**.

We welcome contributions from developers who want to improve the project through bug fixes, features, documentation, and improvements.

---

## Development Setup

Before contributing, please set up the project locally.

### Backend Setup

Ell-ena uses **Supabase** for the backend.

Follow the setup guide:

[BACKEND.md](BACKEND.md)

This guide explains how to:

- Install the Supabase CLI
- Create a Supabase project
- Configure environment variables
- Deploy the database schema
- Configure authentication
- Deploy Edge Functions

### Frontend Setup

The frontend is built using **Flutter**.

Follow the setup guide:

[FRONTEND.md](FRONTEND.md)

This guide explains how to:

- Install Flutter SDK
- Configure development tools
- Install dependencies
- Configure environment variables
- Run the application locally

Ensure both backend and frontend are properly set up before making changes.

## Contribution Workflow

1. **Fork the repository**

2. **Clone your fork**

```bash
git clone https://github.com/YOUR_USERNAME/Ell-ena.git
cd Ell-ena
```

3. **Create a new branch**

```
git checkout -b feature/your-feature-name
```

4. **Make your changes**

5. **Commit your changes**

```
git commit -m "Short description of the change"
```

6. **Push to your fork**

```
git push origin feature/your-feature-name
```

7. **Open a Pull Request**

Submit a Pull Request to the main repository and describe your changes clearly.

---

## Pull Request Guidelines

To help maintain project quality:

- Keep pull requests **small and focused**
- Write **clear commit messages**
- Reference related **GitHub issues**
- Ensure the project builds successfully
- Avoid unrelated changes in the same PR

---

## Reporting Issues

If you find a bug or want to suggest a feature:

1. Check existing issues first  
2. If it doesn't exist, open a new issue  
3. Include clear steps to reproduce the issue
